# DroneWatch

DroneWatch is a multi-agent reinforcement learning project that combines a Rust simulator with a Python training and evaluation stack. The current implementation supports local setup, scripted and random rollouts, PPO training, checkpoint evaluation, Ray Tune sweeps, MLflow logging, and artifact generation.

This site is the guide-focused documentation for the current repository state. It explains how to run the project, how the configuration system works, what the Python training pipeline does, and how to interpret the reports produced by trained policies.

## Start Here

If you want the shortest working path, use this order:

1. Read the Getting Started guide and run `make install`.
2. Run the smoke path with `make rollout-random` and `make ppo-smoke`.
3. Read the Configuration guide before changing YAML or CLI overrides.
4. Read the Training Pipeline guide before modifying PPO or evaluation behavior.
5. Use the Results guide to decide whether a checkpoint is actually good.

## What Exists Today

- Rust simulator exposed through the `swarm_sim` extension and the `dronewatch.sim` wrapper.
- RLlib `MultiAgentEnv` wrapper in `SwarmSearchEnv`.
- Structured YAML configuration composed with OmegaConf and validated with Pydantic.
- Shared-policy PPO training and checkpoint evaluation.
- Random-policy and scripted rollout baselines.
- JSON reports, GIF rendering, and MLflow experiment logging.

## Documentation Scope

This documentation is intentionally focused on the working implementation in this repository today. It does not try to document the full roadmap in `PROJECT_PLAN.md`, and it does not include a large autogenerated API reference.
